# Data Pipeline Implementation Guide (예나 담당)

본 문서는 본 프로젝트에서 **공통 데이터 파이프라인(트래픽 수집 및 전처리)**을 담당하는 구현 가이드이다.  
딥러닝 모델 구현과 학습 로직은 본 문서의 범위에 포함되지 않는다.

---

## 1. 예나 담당 범위 요약

### ✅ 담당
- 트래픽 로그 수집 또는 생성 (실측 / 합성)
- 원본 로그를 입력 텐서 X로 변환
- 공통 입력 규격 `(B, N, T, F)` 보장

### ❌ 비담당
- 학습 타깃(Y) 생성
- 모델 구현 (MLP, LSTM 등)
- 손실 함수, 학습, 평가 로직

---

## 2. 원본 트래픽 로그 포맷 (Collector Output)

Collector는 아래 형식으로 트래픽 로그를 생성하거나 수집한다.

### 2.1 필수 컬럼
| 컬럼명 | 설명 |
|---|---|
| `timestamp` | 트래픽 수집 시각 |
| `node_id` | 노드 식별자 (`node_0`, `node_1`, ...) |
| `bytes` | 해당 interval 동안 전송된 바이트 수 |

> `packets` 컬럼은 선택 사항이며 없어도 동작해야 한다.
