# 주제

경량 딥러닝 모델을 활용한 무선 네트워크 최적화 기법 연구

# 📅 캡스톤 디자인 계획 일정(안)

## 1️⃣ 사전 준비 단계 (방학)

### 📌 1-1. 연구 방향 설정 및 관련 연구 조사

**기간:** 여름방학 초 (1~2주)

- 딥러닝 기반 무선 네트워크 최적화 전반에 대한 선행 연구 조사
- 딥러닝을 활용한 다양한 무선 네트워크 최적화 기법 분석
    
    (예: 트래픽 기반 자원 할당, 혼잡 제어, QoS 최적화 등)
    
- 각 최적화 기법별 적용 가능성 및 특징 비교
- 경량 딥러닝 모델(CNN, LSTM, GRU 등)의 무선 환경 적용 가능성 검토
- 캡스톤 전체 시스템 구조 및 연구 방향 설계

✔ **산출물:**

- 딥러닝 기반 무선 네트워크 최적화 기법 비교 분석 결과
- 최종 연구 대상 기법 선정 근거
- 전체 시스템 아키텍처 초안

### 📌 1-2. 최적화 기법 후보 구현 및 기본 모델 검증

**기간:** 여름방학 중반 (2~3주)

- 1-1 단계에서 도출된 **무선 네트워크 최적화 기법 후보** 중
    
    대표적인 기법을 선정하여 시범 구현
    
    (예: 트래픽 기반 자원 할당 등)
    
- 각 기법에 대해 딥러닝 기반 기본 모델(Base Model) 구현
- (예: 트래픽 데이터 생성 및 전처리 모듈 구현)
- 기본 모델 동작 검증 및 기초 성능 평가

✔ **산출물:**

- 딥러닝 기반 기본 최적화 모델(후보별)
- 기법별 기초 성능 비교 결과
- 최종 적용 기법 선정 자료


### 📌 1-3. 실험 시나리오 및 평가 기준 설계

**기간:** 여름방학 후반 (1~2주)

- 최종 선정된 무선 네트워크 최적화 기법을 중심으로
    
    실험 환경 및 시나리오 설계
    
- (예: 정상 트래픽 및 비정상(공격/이상) 트래픽 상황 정의)
- 성능 및 안정성 평가를 위한 지표 선정
    
    (자원 활용률, 지연 시간, 공정성, 안정성 등)
    
- 경량 딥러닝 모델 적용을 위한 비교 기준 정의

✔ **산출물:**

- 실험 시나리오 정의서
- 성능 및 안정성 평가 지표 목록
- 경량화 비교 실험 기준


## 2️⃣ 연구 결과 정리 및 동계 학술대회 포스터 제작 ( 예상 시기 2월 초)

### 📌 2-1. 방학 중 연구 결과 정리 및 시각화

**기간:** 방학 말 ~ 개강 직전 (1주)

- 방학 중 수행한 딥러닝 기반 무선 네트워크 최적화 기법 연구 결과 정리
- 최종 선정된 최적화 기법 및 기본 모델 구조 정리
- 실험 결과(성능 비교, 트래픽 변화, 자원 할당 결과) 시각화
- 연구의 목적, 방법, 주요 결과 요약

✔ **산출물:**

- 연구 결과 요약 자료
- 실험 결과 그래프 및 도식 자료

---

### 📌 2-2. 동계 학술대회 포스터 제작

**기간:** 방학 종료 전후 (1주)

- 하계 학술대회 제출을 위한 연구 포스터 제작
- 연구 배경, 문제 정의, 제안 기법, 실험 결과 정리
- 지도교수 피드백 반영 및 포스터 수정

✔ **산출물:**

- 하계 학술대회 포스터
- 학술대회 제출 자료

## 3️⃣ 구현 및 고도화 단계 (학기 중)

### 📌 3-1. 경량 딥러닝 모델 설계 및 적용

**기간:** 학기 초 (3~4주)

- 기준 모델 대비 경량 딥러닝 모델 설계
    - 채널 수 축소
    - 레이어 수 감소
    - 파라미터 수 및 연산량 감소
- 경량 딥러닝 모델 학습 및 동작 검증
- 경량화 전·후 성능 비교 실험 수행

✔ **산출물:**

- 경량화 딥러닝 모델
- 모델 복잡도(파라미터 수, 연산량) 비교 결과

---

### 📌 3-2. 경량화 효과 분석 및 최적화 성공 검증

**기간:** 학기 중반 (2~3주)

- 기준 모델과 경량 모델 간 성능 비교
    - 자원 할당 효율
    - 공격 상황에서의 안정성
- 경량화로 인한 성능 저하 여부 분석
- 엣지 및 무선 환경 적용 가능성 평가

✔ **산출물:**

- 성능 비교 그래프
- 경량화 효과 분석 결과

---

## 4️⃣ 정리 및 결과 도출 단계 (학기 후반)

### 📌 4-1. 최종 시스템 통합 및 데모 구현

**기간:** 학기 후반 초 (2주)

- 전체 시스템 통합
- (예: 트래픽 변화에 따른 자원 할당 결과 시각화)
- (예: 정상/이상 트래픽 상황을 포함한 데모 시나리오 구성)

✔ **산출물:**

- 최종 데모 시스템
- 시연 영상 또는 결과 화면

---

### 📌 4-2. 결과 정리 및 캡스톤 산출물 작성

**기간:** 학기 말 (2~3주)

- 캡스톤 최종 보고서 작성
- 발표 자료 및 포스터 제작
- 실험 결과 분석 및 결론 정리

✔ **산출물:**

- 캡스톤 최종 보고서
- 발표 자료 및 포스터
